# Performance Monitoring Integration Plan

## Overview
Integrate server-monitor functionality into llm-proxy as a new "Performance" tab in the admin dashboard. Metrics are collected every 5s, persisted to SQLite, and streamed via WebSocket.

## Architecture

### New crate: `llm-proxy-monitor`
- `src/types.rs` - Snapshot, CpuInfo, RamInfo, GpuInfo, DiskInfo, NetworkInfo, LoadAverage, CpuTemp (uses `time::OffsetDateTime`)
- `src/collector.rs` - collect() function adapted from server-monitor-core
- `src/db.rs` - Database operations (insert, query by time range, cleanup)
- `src/lib.rs` - Re-exports + `MonitorHandle` (broadcast sender) + `spawn_monitor_task()`

### Database changes (`llm-proxy-db`)
- New migration MIGRATION_0002: `system_metrics` table
  - Columns: timestamp, cpu_usage_percent, cpu_cores, ram_*, load_avg_*, gpu_json, disk_json, network_json, cpu_temps_json
  - Index on timestamp
- `src/monitor.rs` - Database methods for system metrics

### Dashboard backend (`llm-proxy-dashboard`)
- `DashboardState` gains `monitor: MonitorHandle`
- `lib.rs`: add `/ws/performance` WebSocket route, start monitor task
- `routes.rs`: add `api_performance_history`, `api_performance_latest`, `ws_performance`
- WebSocket on same port as dashboard (no separate port needed)

### Frontend
- `pages/Performance.tsx` - Stats bar + charts (CPU, RAM, GPU, disk, network)
- Connects to `/ws/performance` WebSocket for live updates
- Fetches `/api/performance/history` on load for historical context
- `App.tsx` - add `/performance` route
- `Layout.tsx` - add "Performance" nav link
- `api.ts` - add Performance types

### Wire up
- `Cargo.toml` (workspace) - add `llm-proxy-monitor`
- `crates/llm-proxy-dashboard/Cargo.toml` - add `llm-proxy-monitor` dependency
- `crates/llm-proxy/src/main.rs` - pass monitor handle to DashboardState

## Collection details
- Interval: 5 seconds
- GPU: nvidia-smi when available, silent skip when not
- CPU temps: /sys/class/thermal when available
- Retention: cleanup snapshots older than 24 hours

## Data size estimate
- ~288 snapshots/hour, ~6912/day
- ~200-500 bytes per row (with JSON blobs)
- ~1.4-3.5 MB/day, manageable for SQLite WAL mode
